# ⚖️ 时间序列分析与预警红线规范

1. **时序切分红线**：
   - 严禁使用 `train_test_split(shuffle=True)`！必须按时间先后顺序（Time-based Split）切分训练集与测试集，绝不允许未来数据泄露。

2. **平稳性与分解要求**：
   - 在建模前，必须对原始时间序列进行 **ADF 单位根检验 (Augmented Dickey-Fuller Test)** 判断平稳性。
   - 必须使用 `statsmodels.tsa.seasonal.seasonal_decompose` 进行 STL 趋势/季节性/残差三要素分解图绘制。

3. **模型对比与外生变量验证**：
   - 必须对比 **基线 SARIMA 模型**（不加搜索指数）与 **SARIMAX 模型**（加入“发烧”搜索指数作为外生变量）。
   - 计算并对比两个模型的 **RMSE（均方根误差）** 与 **MAE（平均绝对误差）**。

4. **强制结果解读与 Insight 预测输出**：
   - 必须在 Notebook 末尾使用 Markdown 输出独立的 **“业务解读与提前预警 Insight”** 章节，明确回答：
     1) 搜索指数相比于医院确诊发病，具备多少天的**提前预警时滞（Lag Time）**？
     2) 模型对未来 3~6 个月的流感爆发峰值与拐点的预判结果是什么？
     3) 给医院药房储备与疾控部门提出了哪些针对性的决策建议？
